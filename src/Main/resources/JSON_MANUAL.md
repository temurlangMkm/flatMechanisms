# JSON Manual для MechIsCool

Этот документ описывает формат JSON-конфигурации механизма, который загружается приложением MechIsCool.

Формат строгий: неизвестные поля запрещены. Если в JSON есть поле, которого нет в этой спецификации, загрузчик выдаст ошибку парсинга.

## 1. Общая структура

```json
{
  "crankSpeed": 2.0,
  "nodes": [],
  "links": []
}
```

| Поле | Тип | Обязательность | Описание |
| --- | --- | --- | --- |
| `crankSpeed` | number | нет | Угловая скорость всех кривошипов, рад/с. По умолчанию `2.0`. |
| `nodes` | array | да | Список узлов механизма. Массив не должен быть пустым. |
| `links` | array | да | Список звеньев механизма. Массив не должен быть пустым. |

## 2. Система координат

- Единицы измерения условные, но должны быть одинаковыми для координат и длин.
- Ось `X` направлена вправо.
- Ось `Y` направлена вниз, как в JavaFX Canvas.
- Точка `{ "x": 0, "y": 0 }` находится в начале мировой системы координат механизма.
- Положительное `crankSpeed` увеличивает фазу кривошипа. Из-за экранной оси `Y` вниз это выглядит как вращение по часовой стрелке.
- Отрицательное `crankSpeed` меняет направление вращения.
- Ползунок угла в интерфейсе задает фазу кривошипа в градусах `0..360`; в JSON скорость задается в радианах в секунду.

## 3. Узлы `nodes`

Каждый узел обязан иметь поля:

| Поле | Тип | Обязательность | Описание |
| --- | --- | --- | --- |
| `id` | string | да | Уникальный идентификатор узла. |
| `type` | string | да | Тип узла: `support`, `joint`, `slider`, `onLink`, `mirrored`. |
| `x` | number | зависит от типа | Координата X. |
| `y` | number | зависит от типа | Координата Y. |
| `assembly` | integer | нет | Ветвь сборки `1` или `2` для неоднозначных пересечений. |

### 3.1. `support`

Неподвижная опора. Координаты обязательны.

```json
{
  "id": "A",
  "type": "support",
  "x": 0,
  "y": 0
}
```

Правила:

- `x` и `y` обязательны.
- Опора считается уже решенной точкой механизма.
- На опору можно ссылаться из `links.from` и `links.to`.

### 3.2. `joint`

Свободный шарнир. Положение вычисляется по звеньям.

```json
{
  "id": "D",
  "type": "joint",
  "assembly": 1
}
```

Правила:

- `x` и `y` необязательны. Если заданы, они используются как начальная подсказка для решателя.
- Если шарнир находится как пересечение двух окружностей, обычно существуют две геометрические точки. Для фиксированного стартового положения задавайте `assembly`.
- Если `assembly` не задан, используется ветвь `1`.

### 3.3. `slider`

Ползун, который движется только вдоль заданной прямой.

```json
{
  "id": "S",
  "type": "slider",
  "x": 180,
  "y": 0,
  "line": {
    "p1": [0, 0],
    "p2": [10, 0]
  },
  "assembly": 1
}
```

Поля:

| Поле | Тип | Обязательность | Описание |
| --- | --- | --- | --- |
| `line` | object | да | Направляющая ползуна. |
| `line.p1` | array[number, number] | да | Первая точка прямой. |
| `line.p2` | array[number, number] | да | Вторая точка прямой. |
| `x`, `y` | number | нет | Начальная подсказка положения ползуна. |
| `assembly` | integer | нет | Выбор одной из двух точек пересечения окружности и прямой. |

Правила:

- `line.p1` и `line.p2` должны содержать ровно два числа.
- `line` задает бесконечную прямую, а не ограниченный отрезок между `p1` и `p2`.
- `p1` и `p2` не должны совпадать по смыслу механизма: иначе направление направляющей неопределенно.
- Для надежного стартового положения задавайте `x/y` или `assembly`.

### 3.4. `onLink`

Точка, жестко закрепленная на существующем звене.

```json
{
  "id": "E",
  "type": "onLink",
  "link": "coupler",
  "distance": 60,
  "orthogonal": -40
}
```

Поля:

| Поле | Тип | Обязательность | Описание |
| --- | --- | --- | --- |
| `link` | string | да | `id` звена, на котором находится точка. |
| `distance` | number | да | Смещение вдоль звена от узла `from` к узлу `to`. |
| `orthogonal` | number | нет | Перпендикулярное смещение. По умолчанию `0`. |

Правила:

- Звено, указанное в `link`, обязано иметь явный `id`.
- `distance = 0` помещает точку в узел `from`.
- `distance = length` помещает точку в узел `to`, если `orthogonal = 0`.
- `distance` может быть больше длины звена или отрицательным: точка окажется на продолжении линии звена.
- `orthogonal` считается влево от направления `from -> to` в экранной системе координат. Отрицательное значение смещает точку вправо от этого направления.

### 3.5. `mirrored`

Точка на продолжении линии `source -> pivot` за точкой `pivot`. Движение точки зависит от положения, скорости и ускорения `source`; `pivot` обычно является опорой.

```json
{
  "id": "C",
  "type": "mirrored",
  "source": "A",
  "pivot": "B",
  "distance": 80
}
```

Поля:

| Поле | Тип | Обязательность | Описание |
| --- | --- | --- | --- |
| `source` | string | да | Исходная точка, от которой строится направление к `pivot`. |
| `pivot` | string | да | Точка, за которой размещается `mirrored`. |
| `distance` | number | да | Положительное расстояние от `pivot` до новой точки. |

Правила:

- `source` и `pivot` должны ссылаться на существующие узлы.
- `distance` задает длину `pivot-C`, а не весь отрезок `source-C`.
- Если `source` и `pivot` совпадают, направление построения не определено.

## 4. Ветвь сборки `assembly`

`assembly` выбирает одну из двух возможных геометрических точек при решении:

- пересечение окружность-окружность для `joint`;
- пересечение окружность-прямая для `slider`.

```json
{
  "id": "D",
  "type": "joint",
  "assembly": 2
}
```

Допустимые значения:

- `1` - первая точка, которую возвращает геометрический решатель;
- `2` - вторая точка.

Если поле не задано, используется `1`. Выбор `assembly` применяется при сбросе механизма и при ручной установке угла кривошипа. Во время анимации решатель дальше выбирает ближайшее положение, чтобы сохранить непрерывность движения.

## 5. Звенья `links`

Каждое звено связывает два узла и задает постоянное расстояние между ними.

| Поле | Тип | Обязательность | Описание |
| --- | --- | --- | --- |
| `id` | string | нет | Уникальный идентификатор звена. Обязателен, если на звено ссылается `onLink`. |
| `type` | string | да | Тип звена: `crank`, `rod`, `rocker`. |
| `from` | string | да | `id` начального узла. |
| `to` | string | да | `id` конечного узла. |
| `length` | number | да | Положительная длина звена. |

### 5.1. `crank`

Ведущий кривошип.

```json
{
  "id": "crank",
  "type": "crank",
  "from": "A",
  "to": "C",
  "length": 40
}
```

Правила:

- `from` обычно должен быть неподвижной опорой или уже решенным узлом.
- `to` вращается вокруг `from`.
- Все кривошипы используют одну общую фазу и один `crankSpeed`.
- Начальный угол кривошипа берется из начальных координат `from` и `to`. Если у `to` нет координат, стартовый угол фактически равен `0`.

### 5.2. `rod`

Обычное жесткое ведомое звено.

```json
{
  "id": "coupler",
  "type": "rod",
  "from": "C",
  "to": "D",
  "length": 120
}
```

### 5.3. `rocker`

Коромысло. В текущей модели решается как жесткое ведомое звено, но рисуется отдельным цветом.

```json
{
  "type": "rocker",
  "from": "B",
  "to": "D",
  "length": 80
}
```

## 6. Ограничения валидации

Загрузчик проверяет:

- JSON не пустой.
- `nodes` и `links` не пустые.
- Все `node.id` уникальны.
- Все непустые `link.id` уникальны.
- Все `links.from` и `links.to` ссылаются на существующие узлы.
- `link.length` задан и больше `0`.
- `link.type` равен только `crank`, `rod` или `rocker`.
- `node.type` равен только `support`, `joint`, `slider`, `onLink` или `mirrored`.
- `support` имеет `x` и `y`.
- `slider` имеет `line.p1` и `line.p2`.
- `onLink` имеет `link` и `distance`.
- `onLink.link` ссылается на существующий явный `link.id`.
- `mirrored` имеет `source`, `pivot` и положительный `distance`.
- `assembly`, если задан, равен только `1` или `2`.

## 7. Минимальный пример четырехзвенника

```json
{
  "crankSpeed": 2,
  "nodes": [
    { "id": "A", "type": "support", "x": 0, "y": 0 },
    { "id": "B", "type": "support", "x": 100, "y": 0 },
    { "id": "C", "type": "joint" },
    { "id": "D", "type": "joint", "assembly": 1 }
  ],
  "links": [
    { "id": "crank", "type": "crank", "from": "A", "to": "C", "length": 40 },
    { "id": "coupler", "type": "rod", "from": "C", "to": "D", "length": 120 },
    { "id": "rocker", "type": "rocker", "from": "B", "to": "D", "length": 80 }
  ]
}
```

## 8. Типичные ошибки

- Ошибка: у `onLink` указан `link`, но у нужного звена нет `id`.
  Решение: добавьте явный `id` этому звену.
- Ошибка: ползун движется по неправильной линии.
  Решение: проверьте `line.p1` и `line.p2`; это направление бесконечной прямой.
- Ошибка: механизм стартует в зеркальной сборке.
  Решение: задайте `assembly: 1` или `assembly: 2` на неоднозначном `joint` или `slider`.
- Ошибка: звено ссылается на несуществующий узел.
  Решение: проверьте точное совпадение строк `id`, `from`, `to`.
- Ошибка: добавлено произвольное поле.
  Решение: удалите поле или добавьте его поддержку в Java-модель.
