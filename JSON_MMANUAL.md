# Руководство по созданию JSON конфигураций механизмов (KinematicVis)

Конфигурации задаются в формате JSON. Каждый механизм состоит из `nodes` (узлов/точек) и `links` (звеньев/стержней).

## Общая структура
```json
{
  "crankSpeed": 2.0,
  "nodes": [ ... ],
  "links": [ ... ]
}
```
- `crankSpeed` — Базовая угловая скорость всех кривошипов (радиан в секунду). По умолчанию `2.0`. Знак определяет направление вращения (положительное = по часовой).

---

## Узлы (Nodes)

Узлы бывают 4 типов. У каждого узла обязательно есть `id` (уникальное имя) и `type`. Опционально можно задать начальные `x` и `y`, чтобы подсказать движку изначальное положение (особенно важно для ползунов, чтобы они не собирались "наизнанку").

### 1. "support" (Опора)
Неподвижная точка, жестко прикрепленная к фону.
```json
{ "id": "A", "type": "support", "x": 0, "y": 0 }
```
- `x`, `y` (обязательно) — статические координаты опоры на холсте.

### 2. "joint" (Шарнир)
Свободный шарнир (точка соединения), чьё положение вычисляется автоматически движком на основе прикрепленных к нему жестких звеньев.
```json
{ "id": "B", "type": "joint" }
```
- Вычислится сам, если к нему подключены известные звенья.

### 3. "slider" (Ползун)
Узел, который может двигаться **только вдоль заданной воображаемой прямой линии**.
```json
{ 
  "id": "C", 
  "type": "slider", 
  "x": 200, 
  "y": 0, 
  "line": { "p1": [0, 0], "p2": [200, 0] } 
}
```
- `line` — направляющая прямая, задается двумя точками `p1` и `p2`, через которые она проходит.
- `x`, `y` (рекомендуется) — примерная стартовая позиция ползуна. Если ползун может оказаться с двух сторон от кривошипа, эта координата помогает выбрать правильную сторону на старте.

### 4. "onLink" (Жестко закрепленный на звене)
Точка, которая "приклеена" к телу любого звена. Отлично подходит для создания треугольных деталей или удлинения концов звена.
```json
{ 
  "id": "D", 
  "type": "onLink", 
  "link": "link1", 
  "distance": 30,
  "orthogonal": 0 
}
```
- `link` (обязательно) — **id** звена (link), к которому крепится эта точка.
- `distance` — расстояние от начала звена (от узла `"from"`) вдоль его линии. Если задать число больше длины самого звена, точка окажется на удлинении звена.
- `orthogonal` (опционально) — перпендикулярное смещение от линии звена влево/вправо. Позволяет создавать вынесенные точки крепления (как у треугольных шатунов).

---

## Звенья (Links)

Звенья соединяют узлы. Вы также можете задать `id` для любого звена (обязательно, если планируете крепить к нему узлы `onLink`).

### 1. "crank" (Кривошип)
Ведущее звено механизма. Оно всегда должно исходить из `support` или другого "решенного" узла, и его конец `"to"` будет вращаться по кругу с постоянной скоростью.
```json
{ "id": "crank1", "type": "crank", "from": "A", "to": "B", "length": 40 }
```
- `length` (обязательно) — радиус (длина) кривошипа.

### 2. "rod" / "rocker" (Шатун / Коромысло)
Ведомое жесткое звено. Расстояние между его концами всегда равно `length`. Движок использует их для решения кинематики всей системы.
```json
{ "id": "rod1", "type": "rod", "from": "B", "to": "C", "length": 120 }
```
- `length` (обязательно) — дистанция между точками `"from"` и `"to"`.

---
## Пример
Если вы хотите прикрепить ползун к середине шатуна, вы создаёте узел `"onLink"` ссылающийся на ID шатуна, и затем соединяете этот новый узел тягой (`"rod"`) с самим ползуном!
