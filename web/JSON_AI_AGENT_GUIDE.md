# Руководство для ИИ-агентов по созданию JSON для MechIsCool

Этот файл предназначен для ИИ-агентов, которые генерируют JSON-конфигурации механизмов для MechIsCool. Следуйте ему буквально: приложение использует строгий Jackson-парсер и не принимает неизвестные поля.

## 1. Цель агента

Нужно создать один валидный JSON-объект:

```json
{
  "crankSpeed": 2.0,
  "nodes": [],
  "links": []
}
```

JSON должен описывать кинематическую схему через узлы и жесткие звенья. Не добавляйте комментарии в JSON. Не добавляйте пояснительный текст внутрь JSON.

Контекст UI: панель `MotionGraph` в приложении адаптивная. Ее высота зависит от viewport, а рендерер сам перераспределяет место между тремя графиками. При изменении UI не закладывайте для этого блока жесткую фиксированную высоту как обязательное условие.

## 2. Фактическая схема

### Root object

Разрешены только поля:

- `crankSpeed`: number, optional, default `2.0`.
- `nodes`: array, required, non-empty.
- `links`: array, required, non-empty.

### Node object

Разрешены только поля:

- `id`: string, required, unique.
- `type`: string, required: `support`, `joint`, `slider`, `onLink`, `mirrored`.
- `x`: number, optional except `support`.
- `y`: number, optional except `support`.
- `line`: object, required only for `slider`.
- `link`: string, required only for `onLink`.
- `source`: string, required only for `mirrored`.
- `pivot`: string, required only for `mirrored`.
- `distance`: number, required only for `onLink`; positive number, required for `mirrored`.
- `orthogonal`: number, optional for `onLink`, default `0`.
- `assembly`: integer, optional, only `1` or `2`.

### Slider line object

Разрешены только поля:

- `p1`: array of exactly 2 numbers.
- `p2`: array of exactly 2 numbers.

### Link object

Разрешены только поля:

- `id`: string, optional, unique if present.
- `type`: string, required: `crank`, `rod`, `rocker`.
- `from`: string, required, existing node id.
- `to`: string, required, existing node id.
- `length`: number, required, must be greater than `0`.

## 3. Система координат и знаки

- Мировая система координат совпадает с экранной логикой JavaFX Canvas.
- `X` растет вправо.
- `Y` растет вниз.
- Все координаты и длины должны быть в одной условной единице.
- `crankSpeed` задается в рад/с.
- Положительный `crankSpeed` увеличивает фазу. При `x = cos(angle)` и `y = sin(angle)` на экране это выглядит как вращение по часовой стрелке.
- Отрицательный `crankSpeed` вращает в обратную сторону.
- Угол UI-ползунка не пишется в JSON; он задает фазу вручную в градусах.

## 4. Алгоритм создания JSON

1. Определите неподвижные точки механизма.
   Создайте для них `support` с обязательными `x` и `y`.

2. Определите ведущий кривошип.
   Создайте `crank` от опоры или уже решенной точки к подвижному узлу. Если подвижный узел не имеет `x/y`, стартовый угол кривошипа будет `0`.

3. Определите остальные подвижные шарниры.
   Создайте `joint`. Если его положение может иметь две сборки, задайте `assembly`.

4. Определите ползуны.
   Создайте `slider` с `line.p1` и `line.p2`. Если есть две возможные позиции на направляющей, задайте `assembly` или стартовые `x/y`.

5. Определите точки на звеньях.
   Создайте `onLink`, только если нужное базовое звено имеет явный `id`.

6. Создайте звенья.
   Все `links.from` и `links.to` должны ссылаться на уже объявленные `nodes.id`.

7. Проверьте граф решаемости.
   Каждый подвижный `joint` обычно должен иметь минимум две связи с уже решаемыми узлами. Каждый `slider` должен иметь хотя бы одну связь с решаемым узлом и свою направляющую.

## 5. Как задавать типы узлов

### `support`

Используйте для неподвижных опор.

```json
{ "id": "A", "type": "support", "x": 0, "y": 0 }
```

Нельзя пропускать `x/y`.

### `joint`

Используйте для свободных шарниров.

```json
{ "id": "D", "type": "joint", "assembly": 1 }
```

`x/y` для `joint` не обязательны. Они полезны как подсказка, если нужно приблизить начальную конфигурацию. Для надежного выбора зеркальной сборки используйте `assembly`.

### `slider`

Используйте для ползуна на прямой.

```json
{
  "id": "S",
  "type": "slider",
  "x": 200,
  "y": 0,
  "line": { "p1": [0, 0], "p2": [10, 0] },
  "assembly": 1
}
```

Важные тонкости:

- `line` задает бесконечную прямую.
- Расстояние между `p1` и `p2` не ограничивает ход ползуна.
- Важны направление и положение прямой, а не длина отрезка `p1-p2`.
- Не задавайте одинаковые `p1` и `p2`.

### `onLink`

Используйте для точки, жестко привязанной к телу звена.

```json
{
  "id": "P",
  "type": "onLink",
  "link": "coupler",
  "distance": 60,
  "orthogonal": -20
}
```

Тонкости:

- `link` - это `id` звена, не `from` и не `to`.
- Базовое звено обязано иметь явный `id`.
- `distance` идет от `link.from` к `link.to`.
- `orthogonal` идет влево от направления `from -> to`; отрицательное значение идет вправо.
- Для простой точки на линии звена используйте `orthogonal: 0` или пропустите `orthogonal`.

### `mirrored`

Используйте для точки на продолжении линии `source -> pivot` за точкой `pivot`.
`distance` задает расстояние от `pivot` до новой точки.

```json
{
  "id": "C",
  "type": "mirrored",
  "source": "A",
  "pivot": "B",
  "distance": 80
}
```

Тонкости:

- `source` - подвижная или решаемая точка, состояние которой задает направление.
- `pivot` - точка, через которую строится продолжение; обычно это опора.
- `distance` - положительное расстояние `pivot-C`, а не длина всего отрезка `source-C`.
- Если `source` и `pivot` совпадают в текущей фазе, направление не определено.

## 6. Как задавать звенья

### `crank`

Ведущее звено:

```json
{ "id": "crank", "type": "crank", "from": "A", "to": "C", "length": 40 }
```

Рекомендации:

- `from` ставьте на `support`.
- `to` обычно `joint`.
- Если нужен начальный угол не `0`, задайте `x/y` у `to` так, чтобы вектор `from -> to` имел нужное направление.

### `rod`

Обычное ведомое жесткое звено:

```json
{ "id": "coupler", "type": "rod", "from": "C", "to": "D", "length": 120 }
```

### `rocker`

Коромысло:

```json
{ "id": "rocker", "type": "rocker", "from": "B", "to": "D", "length": 80 }
```

В расчете `rocker` ведет себя как жесткое ведомое звено. Отличие в основном семантическое и визуальное.

## 7. Выбор ветви `assembly`

Решатель часто получает две точки:

- шарнир как пересечение двух окружностей;
- ползун как пересечение окружности и направляющей прямой.

Используйте:

```json
{ "id": "D", "type": "joint", "assembly": 1 }
```

или:

```json
{ "id": "D", "type": "joint", "assembly": 2 }
```

Практический алгоритм:

1. Сгенерируйте JSON с `assembly: 1`.
2. Если стартовая сборка зеркальная или неправильная, поменяйте на `assembly: 2`.
3. Не используйте произвольные значения: валидны только `1` и `2`.
4. Для нескольких неоднозначных узлов задавайте `assembly` на каждом таком узле отдельно.

## 8. Проверочный список перед отдачей JSON

- JSON является одним объектом, а не массивом.
- Нет комментариев.
- Нет trailing comma.
- Нет неизвестных полей.
- `nodes` не пустой.
- `links` не пустой.
- Все `node.id` уникальны.
- Все ссылки `from`, `to`, `link` написаны точно как соответствующие `id`.
- Все `link.length` больше `0`.
- У каждого `support` есть `x/y`.
- У каждого `slider` есть `line.p1/p2`.
- У каждого `onLink` есть `link` и `distance`.
- У каждого звена, на которое ссылается `onLink`, есть явный `id`.
- Для неоднозначных стартовых положений задан `assembly`.

## 9. Шаблон: четырехзвенник

```json
{
  "crankSpeed": 2,
  "nodes": [
    { "id": "A", "type": "support", "x": 0, "y": 0 },
    { "id": "B", "type": "support", "x": 100, "y": 0 },
    { "id": "C", "type": "joint" },
    { "id": "D", "type": "joint", "assembly": 1 }
  ],
  "links": [
    { "id": "crank", "type": "crank", "from": "A", "to": "C", "length": 40 },
    { "id": "coupler", "type": "rod", "from": "C", "to": "D", "length": 120 },
    { "id": "rocker", "type": "rocker", "from": "B", "to": "D", "length": 80 }
  ]
}
```

Если точка `D` стартует с неправильной стороны, поменяйте `assembly` на `2`.

## 10. Шаблон: кривошипно-ползунный механизм

```json
{
  "crankSpeed": 2,
  "nodes": [
    { "id": "A", "type": "support", "x": 0, "y": 0 },
    { "id": "B", "type": "joint" },
    {
      "id": "S",
      "type": "slider",
      "x": 180,
      "y": 0,
      "line": { "p1": [0, 0], "p2": [10, 0] },
      "assembly": 1
    }
  ],
  "links": [
    { "id": "crank", "type": "crank", "from": "A", "to": "B", "length": 50 },
    { "id": "rod", "type": "rod", "from": "B", "to": "S", "length": 130 }
  ]
}
```

## 11. Шаблон: точка на звене

```json
{
  "crankSpeed": 2,
  "nodes": [
    { "id": "A", "type": "support", "x": 0, "y": 0 },
    { "id": "B", "type": "support", "x": 100, "y": 0 },
    { "id": "C", "type": "joint" },
    { "id": "D", "type": "joint", "assembly": 1 },
    { "id": "P", "type": "onLink", "link": "coupler", "distance": 60, "orthogonal": -30 }
  ],
  "links": [
    { "id": "crank", "type": "crank", "from": "A", "to": "C", "length": 40 },
    { "id": "coupler", "type": "rod", "from": "C", "to": "D", "length": 120 },
    { "id": "rocker", "type": "rocker", "from": "B", "to": "D", "length": 80 }
  ]
}
```

Не добавляйте фиктивные звенья от `P`, если точка нужна только для отображения и расчета ее скорости/ускорения как точки тела звена.

## 11.1. Шаблон: mirrored-точка за опорой

```json
{
  "crankSpeed": 2,
  "nodes": [
    { "id": "O", "type": "support", "x": 0, "y": 0 },
    { "id": "B", "type": "support", "x": 100, "y": 0 },
    { "id": "A", "type": "joint", "x": 40, "y": 0 },
    { "id": "C", "type": "mirrored", "source": "A", "pivot": "B", "distance": 80 }
  ],
  "links": [
    { "id": "crank", "type": "crank", "from": "O", "to": "A", "length": 40 }
  ]
}
```

## 12. Частые ошибки ИИ-генерации

- Добавление полей вроде `name`, `mass`, `color`, `angle`, `fixed`, `initialAngle`.
  Эти поля не поддерживаются.
- Использование `joint` вместо `support` для неподвижной точки.
  Неподвижная точка должна быть `support`.
- Ссылка `onLink.link` на узел, а не на звено.
  Нужно ссылаться на `link.id`.
- Описание слайдера как отрезка.
  В MechIsCool это бесконечная направляющая прямая.
- Создание нерешаемого узла с одной связью и без направляющей.
  `joint` обычно требует две геометрические связи.
- Использование `assembly: "1"` строкой.
  Нужно число: `assembly: 1`.
