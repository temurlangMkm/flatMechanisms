Краткое описание

Визуализатор плоских механизмов, реализованный на языке Processing. Цель — интерактивное отображение и исследование различных типов плоских механизмов (звенья, шарниры, поводки и т.д.) с подстройкой параметров через JSON-вход.

Основные возможности (планируемые)

Загрузка описания механизма в формате JSON.

Отображение звеньев и соединений в 2D.

Анимация движения при заданных приводах/ограничениях.

Визуализация кинематических траекторий и точек.

Возможность отладки и пошагового исполнения.
Формат входных данных

Входной файл задаётся в формате JSON. Корневой объект содержит описание механизма.

Общая структура
{
  "mechanism": {
    "project_name": "string",
    "links": [ ... ],
    "joints": [ ... ]
  }
}
Описание звеньев (links)

Каждое звено содержит:

id — уникальный идентификатор звена

type — тип звена (Ground, Crank, Coupler, Rocker и др.)

length — длина звена (float)

fixed — (опционально) фиксированное звено

input_angle — (для ведущего звена) начальный угол в градусах

Пример:

{
  "id": "B",
  "type": "Crank",
  "length": 50.0,
  "input_angle": 45.0
}
Описание соединений (joints)

Каждое соединение содержит:

type — тип соединения (например, Revolute)

pair — массив из двух идентификаторов соединяемых звеньев

origin — (опционально) координата закрепления в системе (для Ground)

Пример:

{
  "type": "Revolute",
  "pair": ["A", "B"],
  "origin": [0, 0]
}
Пример: четырёхзвенник

Ниже приведён пример кривошипно-коромыслового механизма:

A — основание (Ground)

B — кривошип (Crank)

C — шатун (Coupler)

D — коромысло (Rocker)

Механизм замкнут четырьмя вращательными шарнирами (Revolute).

(Ваш предоставленный JSON может быть размещён в папке data/ как пример конфигурации.)

Запуск

Установить Processing IDE.

Открыть главный .pde файл проекта.

Поместить JSON-файл в папку data/.

Запустить sketch через Processing.

Точные инструкции будут обновлены после стабилизации структуры.
